# Password-protected Link Penting page with accordion product directory

This change adds a new page at `/public/pages/linkpenting.html` — a password-gated directory of Google Drive and YouTube resources for 14 Manulife insurance products, organized as expanding accordion cards. The page targets "nasabah dan tim Project Cuan" (as distinct from the agent-only tools page). It follows the same client-side auth pattern and visual system as agen-tools.html. Navigation links are wired into the header mobile menu, footer, and index.html bottom section.

Watch for: The source document contains Sections II and III (Financing/Keagenan, Starter Kit) with ~63 additional links that were not included — **likely** intentional since that content already lives in agen-tools.html, but worth confirming this was a deliberate scope decision and not an oversight. The protected content is present in the DOM before authentication (hidden by `display:none`) — **confirmed** — meaning all 112 external links are visible in page source without the password.

## High-level view

The page is a single 759-line HTML file containing inline CSS, an auth gate, a quick-nav pill bar, and 14 accordion sections for each product (MIUHC through MiAction). Each accordion reveals a list of Drive/YouTube links and a "Folder Lengkap" button. The content was extracted from the converted PDF source file — specifically Section I (Simple Produk Knowledge) which contains the product-specific resources.

The auth mechanism is the same `sessionStorage` + plaintext password comparison used by agen-tools.html. The password provides a UI-level gate, not a security boundary. The actual protection of the underlying resources relies on Google Drive sharing permissions.

Navigation integration touches three files — index.html gains a second pill button in the AGEN TOOLS LINK bar (container changed from `text-align:center` to `display:flex` to accommodate two items), header.html gets a mobile-menu entry after Agen Tools, and footer.html gets a link after Khusus Agen.

## Content extraction from source PDF

The source file (`Manulife File Penting - Directory Per 02 Juni 2026 (2)-converted.html`) contains three top-level sections: I. Simple Produk Knowledge (14 products, ~138 raw href instances including duplicates), II. Financing/Keagenan/Kompensasi, and III. Starter Kit & New Agent Tools. The implementation extracts only Section I, yielding 112 unique external links across the 14 product accordions plus the folder-level links.

Sections II and III map directly to content already present in agen-tools.html (Financing, Keagenan & AAJI, Kompensasi, Starter Kit sections). The scope decision appears intentional — linkpenting.html is for product knowledge consumed by clients and team members, while agen-tools.html holds operational tools for agents. However, the page's subtitle says "Perpustakaan Digital BMI Manulife" without qualification, which could confuse users expecting the full directory.

## Auth gate and content visibility

The content div (`#protectedContent`) is in the DOM but hidden with inline `style="display:none"` and revealed by JS on successful auth. All 112 Google Drive/YouTube URLs are fully readable in page source without the password. If the linked resources use "anyone with the link" sharing, the password gate is purely cosmetic. This is the established pattern (agen-tools.html does the same), but the risk scales with the number of links exposed — this page has twice as many external links as agen-tools.

The auth gate doesn't clear the error message on subsequent input — once shown, `authError` stays visible until successful login redirects. Minor UX gap.

## Accordion interaction

The accordion uses `max-height: 2000px` on `.active` for expand animation. The largest section (MIUHC) has 14 items — well within that ceiling now, but if sections grow significantly (e.g., additional sub-items added later), the fixed max-height could clip content without any visible indication that content is hidden.

## Navigation integration in index.html

The index.html change converts the AGEN TOOLS LINK container from `text-align:center` to `display:flex;justify-content:center;flex-wrap:wrap;gap:var(--sp-3)` to accommodate two pill buttons. This is a layout change that affects the existing Agen Tools link positioning — on narrow viewports the buttons now wrap vertically rather than appearing as inline-block elements.
