# Password-protected agent tools page with commission calculator

A new 781-line HTML page at `/public/pages/agen-tools.html` consolidates agent-facing resources behind a client-side password gate. The page aggregates five resource sections (Financing, Keagenan & AAJI, Kompensasi, Manu Pro/MABC/Billionaire, Starter Kit) and copies the five commission tables from `komisi-agen.html`. It follows the established site patterns: dark-mode inline script, CSS variable usage, shared header/footer via main.js.

**Watch for:** Many individual resource links are placeholders pointing to the same parent folder rather than specific documents (confirmed). The commission tables were copied rather than moved, creating a maintenance duplication concern (confirmed). All "Submit Tools" and "Recruiting Tools" items link to a generic GitHub directory rather than actual resource files (confirmed).

## High-level view

The page structure and auth pattern clone `komisi-agen.html`, using a separate sessionStorage key (`pc_agen_tools_auth`). The password sits in plaintext in the JS source and all protected content ships in the HTML — same pattern as the existing komisi page.

Content sections 1-3 (Financing, Keagenan, Kompensasi) pull real Google Drive links from `Linkweb2.html`, though several items with distinct labels all resolve to the same root folder URL. Section 4 (Manu Pro/MABC/Billionaire) has all six items pointing to the Keagenan folder — the source had no links for this section, so these are stubs.

The Starter Kit section splits between a few real Drive links (Selling Tools A-F) and nine items pointing to `github.com/.../Source/Agency/`. These won't serve actual documents — agents will land on a GitHub source tree.

The commission calculator duplicates all five tables from `komisi-agen.html` rather than removing them from the original. The user instruction says "pindahkan" (move), implying removal from the source.

## Placeholder links masquerading as distinct resources

In the Financing section, items D (Form Financing), F (Memo Reguler Financing), and G (Lain Lain nya) all resolve to `https://drive.google.com/drive/folders/1J7p5f6UzA3ZFfSU1jTxSCJC29FYCPjP6` — the same folder used by the "Buka Folder" button. The source file `Linkweb2.html` only had four distinct links in the Financing section (Materi, Next Pro, Skema Special, Kalkulator), so D/F/G were expanded from the user spec's content outline without corresponding distinct destinations.

The same pattern is more severe in section 4: all six items A-F for Manu Pro, MABC, and Billionaire point to the Keagenan & AAJI parent folder (`1PWI2oTjSmlQgVeqetmKrZCzdZm0cDBK-`). The original source (`Manulife File Penting` HTML) lists these items without href attributes — they were text-only placeholders. The implementation linked them all to the nearest parent folder rather than leaving them as non-link text or noting them as "coming soon."

## GitHub links as resource destinations

Nine items in the Starter Kit section link to `https://github.com/petrusjakub/Project-Cuan-3.0/blob/main/Source/Agency/`. This URL shows a GitHub source tree. For an end user (an insurance agent), this is confusing — they'd expect a Google Drive document or downloadable PDF. The user instruction did reference this GitHub folder for "Lain Lain" items specifically, but the implementation extended it to all Submit Tools and Recruiting Tools items as well.

## Commission table duplication

The user instruction says "pindahkan kalkulator income ke page ini" (move the income calculator to this page). The implementation copies the tables but doesn't remove them from `komisi-agen.html` — the diff shows zero modifications to that file. The word "pindahkan" implies removal from the source, so `komisi-agen.html` should either have its tables removed or redirect to the new page's calculator section.
