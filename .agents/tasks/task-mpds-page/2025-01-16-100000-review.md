# MPDS product page with interactive benefit calculator

A complete product landing page for Manulife Perlindungan Diri Syariah (MPDS), following the same architectural pattern as mdla.html and mdwa.html: hero section, advantage cards, benefit cards, eligibility grid, premium table, interactive calculator, FAQ accordion, and CTA. The calculator computes death benefit (santunan), accident benefit (+50%), total contribution, and maturity return based on MPK (5/12 years), payment mode, entry age, and contribution amount.

**Watch for:** The calculator only supports bulanan/tahunan modes while the page's own eligibility section and the Sample reference both list four payment modes including kuartalan and semesteran (confirmed). The premium table "UP Kecelakaan (Total)" column label may confuse readers who interpret it as only the accident-specific add-on rather than the combined death+accident payout (possible).

## High-level view

The calculator arithmetic is correct: monthly equivalent conversion (÷12 for tahunan), correct multipliers (250/450), Rp 1.2B accident cap applied to the +50% portion only, and age-differentiated return percentages. Premium table values all verify against the formulas.

The calculator only supports two of four payment modes. The eligibility section on this very page states "Bulanan, Kuartalan, Semesteran, atau Tahunan," and the Sample/Mpds.html reference calculator implements all four. Prospects who prefer quarterly or semiannual payment have no way to estimate their benefits.

Custom input for tahunan mode doesn't enforce multiples of 12, which can produce fractional monthly equivalents (e.g., Rp 3,500,000/year → 291,666.67/month) displayed with decimals. Not a bug, but a UX roughness.

## Calculator edge case: non-integer monthly equivalents

When mode is "tahunan," the formula divides by 12:

```javascript
kontribusiBulanan = kontribusiInput / 12;
```

All tahunan presets (3M, 3.6M, 6M, 9M, 12M, etc.) are divisible by 12, so presets are safe. But the custom input path only enforces a minimum of Rp 3,000,000 — it doesn't enforce divisibility. A user entering Rp 3,500,000 yearly would see `Santunan = 450 × Rp 291.666,67` displayed with decimals. Consider either rounding to the nearest integer or restricting custom input to multiples of 12,000 in tahunan mode.

## Missing payment modes vs. source material

The Sample/Mpds.html reference calculator includes kuartalan and semesteran modes with their own conversion logic. The new page's eligibility section explicitly states: "Metode: Bulanan, Kuartalan, Semesteran, atau Tahunan." The calculator's mode selector only shows two options. For a sales-oriented page, this creates an internal inconsistency and a functional gap for prospects preferring quarterly/semiannual billing.
